## Обзор

Chimera — современный настольный прокси-клиент для пользователей, которым нужен стабильный, понятный и поддерживаемый опыт работы на Windows, macOS и Linux/NixOS. Проект построен на Tauri, Rust и React, сохраняя лёгкость и отзывчивость настольного приложения и одновременно обеспечивая надёжную поддержку прокси-ядра, подписок, системного прокси, сервисного режима и обновлений.

Если вам нужен инструмент для повседневного использования, Chimera не пытается усложнить интерфейс. Основная цель — сделать обычные действия проще: импорт подписок, переключение узлов, просмотр соединений, настройка правил, управление ядрами и обновления должны быть прямыми, понятными и не мешать работе.

## Почему стоит выбрать Chimera

- Кроссплатформенный настольный опыт: внимание к реальным сценариям на Windows, macOS и Linux/NixOS.
- Поддержка нескольких ядер: основной фокус на запуске, переключении, обновлении и управлении такими ядрами, как `chimera-client`, `clash-rs` и `mihomo`.
- Полная поддержка протоколов: ориентир на распространённые сочетания и практическую совместимость ядер.
- Полноценная консоль управления прокси: управление подписками и профилями, просмотр групп и узлов, наблюдение за соединениями и настройка runtime-конфигурации.
- Поддержка сервисного режима: подходит для пользователей, которым нужен более безопасный фоновый запуск, системные прокси-возможности и стабильная длительная работа.
- Удобство для диагностики: предоставляет доступ к журналам, соединениям, правилам, каталогу конфигурации и каталогу ядер, снижая стоимость поиска проблем.

## Поддерживаемые протоколы

Сейчас Chimera сосредоточен на распространённых сочетаниях прокси и совместимости ядер, включая:

- `trojan + ws + tls`
- `reality + tcp`
- `hysteria2`
- `xhttp`

## Основные возможности

- Управление подписками и профилями: импорт, обновление, удаление, просмотр и редактирование удалённых конфигураций.
- Управление прокси-узлами: просмотр групп прокси, выбор узлов и синхронное обновление runtime-состояния после переключения.
- Runtime-конфигурация: чтение и настройка конфигурации Clash для диагностики и тонкой настройки.
- Управление ядром: просмотр состояния ядра, переключение ядра, перезапуск sidecar и получение или обновление версий ядра.
- Системная интеграция: чтение системного прокси, поддержка deep link, запуска в одном экземпляре, уведомлений, диалогов и управления процессами.
- Сервисный режим: установка, удаление, запуск, остановка и перезапуск службы.
- Поддержка обновлений: проверка обновлений приложения и отдельная цепочка управления обновлением версий ядра.

## Частые команды для разработки

Установить зависимости:

```bash
pnpm install
```

Подготовить метаданные ядер:

```bash
pnpm generate:manifest
```

Подготовить бинарные ресурсы ядер и выполнить проверку:

```bash
pnpm check
```

Запустить настольное приложение:

```bash
pnpm dev:diff
```

Собрать приложение:

```bash
pnpm build
```

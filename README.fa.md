## معرفی

Chimera یک کلاینت پراکسی دسکتاپ مدرن است برای کاربرانی که روی Windows، macOS و Linux/NixOS به یک تجربه پایدار، شفاف و قابل نگه‌داری نیاز دارند. این پروژه با Tauri، Rust و React ساخته شده و در عین حفظ سبکی و سرعت واکنش‌پذیری یک برنامه دسکتاپ، از هسته‌های پراکسی، پروفایل‌های اشتراک، پراکسی سیستم، حالت سرویس و به‌روزرسانی‌ها به‌صورت قابل اتکا پشتیبانی می‌کند.

اگر به دنبال ابزاری برای استفاده روزمره هستید، هدف Chimera پیچیده کردن رابط کاربری نیست. تمرکز آن این است که مسیرهای رایج مثل وارد کردن اشتراک، جابه‌جایی بین نودها، مشاهده اتصال‌ها، تنظیم قوانین، مدیریت هسته‌ها و رسیدگی به به‌روزرسانی‌ها تا حد ممکن مستقیم، قابل فهم و کم‌مزاحمت باشند.

## چرا Chimera

- تجربه دسکتاپ چندسکویی: تمرکز روی مسائل واقعی استفاده در Windows، macOS و Linux/NixOS.
- جهت‌گیری در پشتیبانی از چند هسته: تمرکز اصلی روی اجرا، جابه‌جایی، به‌روزرسانی و مدیریت هسته‌هایی مثل `chimera-client`، `clash-rs` و `mihomo`.
- پشتیبانی کامل از پروتکل‌ها: با تمرکز بر ترکیب‌های رایج و سازگاری عملی هسته‌ها.
- کنسول کامل مدیریت پراکسی: مدیریت اشتراک‌ها و پروفایل‌ها، مشاهده گروه‌های پراکسی و نودها، بررسی وضعیت اتصال‌ها و تنظیم پیکربندی زمان اجرا.
- پشتیبانی از حالت سرویس: مناسب برای کاربرانی که اجرای پس‌زمینه پایدارتر، قابلیت‌های پراکسی در سطح سیستم و استفاده بلندمدت می‌خواهند.
- مناسب برای عیب‌یابی: دسترسی به لاگ‌ها، اتصال‌ها، قوانین، مسیر فایل‌های پیکربندی و مسیر هسته‌ها برای کاهش هزینه پیدا کردن مشکل.

## پروتکل‌های پشتیبانی‌شده در حال حاضر

در حال حاضر Chimera روی ترکیب‌های رایج پراکسی و سازگاری هسته‌ها تمرکز دارد، از جمله:

- `trojan + ws + tls`
- `reality + tcp`
- `hysteria2`
- `xhttp`

## قابلیت‌های اصلی

- مدیریت اشتراک و پروفایل: وارد کردن، به‌روزرسانی، حذف، مشاهده و ویرایش پیکربندی‌های راه‌دور.
- مدیریت نودهای پراکسی: مشاهده گروه‌های پراکسی، انتخاب نود و نوسازی وضعیت زمان اجرا پس از جابه‌جایی.
- پیکربندی زمان اجرا: خواندن و تنظیم پیکربندی‌های مرتبط با Clash برای عیب‌یابی و تنظیم دقیق.
- مدیریت هسته: مشاهده وضعیت هسته، جابه‌جایی هسته، راه‌اندازی مجدد sidecar و دریافت یا به‌روزرسانی نسخه هسته.
- یکپارچه‌سازی با سیستم: خواندن پراکسی سیستم، پشتیبانی از deep link، اجرای تک‌نمونه، اعلان‌ها، دیالوگ‌ها و مدیریت فرایند.
- حالت سرویس: نصب، حذف، اجرا، توقف و راه‌اندازی مجدد سرویس.
- قابلیت به‌روزرسانی: بررسی به‌روزرسانی برنامه و نگه داشتن یک مسیر مستقل برای مدیریت به‌روزرسانی نسخه هسته.

## دستورهای رایج توسعه

نصب وابستگی‌ها:

```bash
pnpm install
```

آماده‌سازی اطلاعات هسته:

```bash
pnpm generate:manifest
```

آماده‌سازی فایل‌های دودویی هسته و اجرای بررسی‌ها:

```bash
pnpm check
```

اجرای برنامه دسکتاپ:

```bash
pnpm dev:diff
```

ساخت برنامه:

```bash
pnpm build
```
